Dans un projet dev, on a deux parties: le bac et le front.

Les deux equipes peuvent ben travailler en parallelle car on a deja definit le cahier de charge.

Maintenant, au debut ou le backend n'est pas encore terminer,pour la partie front, il faut simuler ce que l'on doit avoir apres :
1- Pour cela , on va cree un fichier app/data.ts qui va jouer le role de base de donnees

Maintenant, pour pouvoir utiliser les models crees, il faut ajouter le mot cle
export qui est un peu l'equivalent de public.

exemple: Nous on avait cree le model comme suit:

dans ce cas, le model ne peut etre utilise que dans le fichier data.ts
interface Classe{
  id?:string; //le point d'interrogation signifie que c'est pas obligatoire
  code:string;
  nom:string;
  description?:string;

}

Maintenant, pour pouvoir l'utiliser dans d'autres fichiers, on ajoute export comme suit:

export interface Classe{
  id?:string; //le point d'interrogation signifie que c'est pas obligatoire
  code:string;
  nom:string;
  description?:string;

}

Maintenant, dans data.ts, on une liste d'etudiants:

const ETUDIANTS:Etudiant[]=[

]
NB: Generalement, quand ona une relation comme notre cas entre etudiant et classe, alors, on met
la relation en dernier(Voir l'enregistrement dans etudiants)

Maintenant, creeons un autre composant pour pouvoir gerer la navigation entre
composant.

Lorsque tous nos pages ont le meme entete et le meme footer, alors, on peut ajouter dans
app.component.html: les balises footer et header
on met le  header  en haut , le footer en bas et le router outlet qui depends
de la page ou on est, on le met au milieu.

Si au niveau d'une autre page, on doit pas avoir l'entete, par exemple la page
loggin, alors, on laisse seulement la balise <router-outlet/>.

Si on oblige a toute les pages d'avoir le meme footer, ci  est generalement le cas:
On ajoute la balise <footer><footer/> en bas de celui de router-outlet.
Pour dir que toute les pages ont le meme footer, chacune customise son entete

router-outlet est une variable qui  est egal au composant mis dans app.route  dont le path est mis a vide:

ie:
routes[
{path:'' , component:"Component "}

]
